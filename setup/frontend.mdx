---

The frontend is a **Next.js 16** application using **React 19** and **Tailwind CSS 4**. It communicates with the FastAPI backend through the `NEXT_PUBLIC_AGENT_API` environment variable.

## Prerequisites

- Node.js 18 or later
- npm, yarn, or pnpm

## Run locally

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Sai3Karthi/Thirukkural.AI.git
    cd Thirukkural.AI
    ```
  </Step>

  <Step title="Move into the frontend directory">
    ```bash
    cd frontend_new
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```

    </CodeGroup>
  </Step>

  <Step title="Set the backend URL">
    Create a `.env.local` file in `frontend_new/` and set the backend base URL:

    ```bash .env.local
    NEXT_PUBLIC_AGENT_API=http://localhost:8000
    ```

    <Note>
      `NEXT_PUBLIC_AGENT_API` must be prefixed with `NEXT_PUBLIC_` so Next.js exposes it to the browser bundle. Without this variable the AI features will not work — the client falls back to `http://localhost:8000` but requests will fail if the backend is running on a different host or port.
    </Note>
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    The app opens at [http://localhost:3000](http://localhost:3000).
  </Step>
</Steps>

## Available npm scripts

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `next dev` | Start the development server with hot reload |
| `build` | `next build` | Produce an optimised production build |
| `start` | `next start` | Serve the production build |
| `lint` | `eslint` | Run ESLint across the project |

## Production build

```bash
npm run build
npm start
```

<Tip>
  [Vercel](https://vercel.com) is the recommended deployment platform for Next.js. It supports automatic builds from your Git branch, environment variable management, and zero-config Next.js optimisations. Set `NEXT_PUBLIC_AGENT_API` to your deployed backend URL in the Vercel project settings.
</Tip>

## Quiz answer persistence

Quiz answers and the derived personality profile are persisted in the browser's **localStorage**. This means the quiz state survives page refreshes without requiring a backend session. Clearing browser storage or opening a private window resets the quiz.
