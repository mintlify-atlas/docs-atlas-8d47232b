## The user flow

Thirukkural.AI has three stages: a one-time personality quiz, an emotion search, and the AI response. This section walks through each stage in the browser.

    ### How the scoring works

    Each answer adds points to one or more of five personality dimensions. The dimension with the highest total determines your type.

    | Dimension | Description |
    |-----------|-------------|
    | `wisdom` | Analytical and ethical reasoning |
    | `compassion` | Empathy, relationships, and kindness |
    | `strength` | Courage, discipline, and leadership |
    | `harmony` | Balance, forgiveness, and inner peace |
    | `ego` | Ambition, pride, and status-seeking |

    The ego dimension has an **override rule**: if your `ego` score equals the maximum across all dimensions, *or* if it exceeds 5 points and is greater than or equal to your `strength` score, you are assigned **The Ambitious Achiever** regardless of other dimension totals.

    <Tip>
      Answer honestly rather than ideally. The personality type shapes every subsequent Kural recommendation, so a more accurate profile produces more resonant results.
    </Tip>

  <Step title="View your personality profile">
    After finishing the quiz, the landing page displays your personality type, a short description, and a set of trait tags. For example:

    > **Your Personality: The Wise Seeker**
    > You value knowledge, truth, and ethical decision-making.
    > Tags: Analytical · Ethical · Thoughtful

    Your profile is passed with every API request so the AI can tailor the tone, emphasis, and choice of Kural to match your archetype.
  </Step>

  <Step title="Search by emotion">
    Type a free-form description of how you feel into the search bar — for example:

    - *"I'm feeling overwhelmed at work and don't know where to start"*
    - *"I had an argument with a close friend and I feel guilty"*
    - *"I've been putting off an important decision for weeks"*

    Alternatively, click one of the pre-set emotion chips below the search bar to pre-fill the query.

    Then click **Get My Kural**.

    <Note>
      The search bar accepts plain English. You do not need to know any Tamil or have read the Thirukkural before.
    </Note>
  </Step>

  <Step title="Use Surprise Me">
    Click the **Surprise Me** button to receive a random Kural matched to your personality type without typing a query. This uses a curated set of verses selected per personality archetype — a good way to explore the text without a specific emotional context in mind.
  </Step>

  <Step title="Read your Kural response">
    The AI response panel contains three sections:

    **Summary** — A one-sentence headline that frames your query in the context of the retrieved Kural wisdom.

    **Kurals** — One or more matching verses, each showing:
    - Kural number, virtue category, and theme
    - Tamil text in the traditional 4+3 word format
    - Full English translation

    **Actionable guidance** — A set of labeled, practical steps derived from the Kural's teaching and tailored to your personality. Each entry has a short label (e.g., *immediate action*, *reflection*, *long-term mindset*) and a prose explanation.

    Click **View Analysis Details** to expand the planner panel, which shows the AI's situation summary, emotional tone assessment, and the list of retrieved Kural IDs with their relevance scores.
  </Step>

## Calling the API directly

The backend exposes a single inference endpoint. No authentication is required in the default local setup.

<Steps>
  <Step title="Start the backend">
    ```bash
    cd backend/mainModules
    uvicorn api_server:app --host 0.0.0.0 --port 8000 --reload
    ```

    Verify the server is healthy:

    ```bash
    curl http://localhost:8000/health
    ```

    ```json
    { "status": "ok" }
    ```
  </Step>

  <Step title="Send a request with curl">
    The endpoint accepts a `question` string and an optional `personality` object.

    ```bash
    curl --request POST \
      --url http://localhost:8000/api/v1/ask \
      --header 'Content-Type: application/json' \
      --data '{
        "question": "I feel overwhelmed and don'\''t know where to start",
        "personality": {
          "type": "The Wise Seeker",
          "description": "You value knowledge, truth, and ethical decision-making",
          "traits": ["Analytical", "Ethical", "Thoughtful"]
        }
      }'
    ```
  </Step>

  <Step title="Parse the response">
    A successful response has the shape defined by `AskResponse` in `api_server.py`:

    ```json
    {
      "question": "I feel overwhelmed and don't know where to start",
      "response": {
        "summary": "When the path ahead feels unclear, Thiruvalluvar counsels turning inward before acting outward.",
        "kurals": [
          {
            "kural_id": 423,
            "tamil": "எண்ணிய எண்ணியாங்கு எய்துப எண்ணியார்\nதிண்ணிய ராகப் பெறின்",
            "english": "Those who think clearly and with firm resolve will achieve exactly what they set their minds to.",
            "virtue": "The possession of steadiness",
            "theme": "Decisiveness"
          }
        ],
        "actions": {
          "immediate_action": "Write down the single most important task. Do only that today.",
          "reflection": "Consider what Kural 423 means: resolve precedes achievement. Clarity is not a precondition — it is a result of committing to a direction.",
          "long_term_mindset": "Build a daily reflection practice. The Wise Seeker archetype benefits most from structured self-assessment."
        }
      },
      "raw_response": "...",
      "plan": {
        "situation_summary": "User is experiencing decision paralysis.",
        "emotional_tone": "anxious",
        "likely_needs": ["prioritization", "reassurance", "practical first step"],
        "action_outline": ["Identify root cause of overwhelm", "Narrow focus to one action", "Build momentum from small wins"]
      },
      "retrieved_kurals": [
        { "kural_id": 423, "score": 4.3821 },
        { "kural_id": 611, "score": 2.9104 }
      ]
    }
    ```

    <Note>
      The `response` field is a parsed JSON object extracted from the model's text output. `raw_response` contains the unprocessed model string if you need to debug or re-parse it.
    </Note>
  </Step>

  <Step title="Call from TypeScript">
    The frontend's `fetchAgentResponse` function in `src/utils/agentClient.ts` wraps the API call and accepts an optional `personality` parameter:

    ```typescript
    import { Personality } from './quizLogic';

    const DEFAULT_API_BASE = "http://localhost:8000";

    const normalizeBaseUrl = (url: string | undefined): string => {
      if (!url) return DEFAULT_API_BASE;
      return url.endsWith("/") ? url.slice(0, -1) : url;
    };

    export interface AgentResponse {
      response?: any;
      question?: string;
      plan?: any;
      retrieved_kurals?: any[];
      raw_response?: string;
    }

    export async function fetchAgentResponse(
      question: string,
      personality?: Personality,
      signal?: AbortSignal
    ): Promise<AgentResponse> {
      const sanitized = question?.trim();
      if (!sanitized) {
        throw new Error("Question cannot be empty");
      }

      const baseUrl = normalizeBaseUrl(process.env.NEXT_PUBLIC_AGENT_API);
      const response = await fetch(`${baseUrl}/api/v1/ask`, {
        method: "POST",
        headers: {
          "Content-Type": "application/json",
        },
        body: JSON.stringify({ question: sanitized, personality }),
        signal,
      });

      if (!response.ok) {
        const errorPayload = await response.json().catch(() => ({}));
        const detail = errorPayload?.detail || response.statusText;
        throw new Error(detail || "Agent request failed");
      }

      return response.json();
    }
    ```

    Set the `NEXT_PUBLIC_AGENT_API` environment variable in your `.env.local` to point to your backend URL. If the variable is absent, the client defaults to `http://localhost:8000`.

    ```bash
    # .env.local
    NEXT_PUBLIC_AGENT_API=http://localhost:8000
    ```
  </Step>
</Steps>

### AI response structure

| Field | Type | Description |
|-------|------|-------------|
| `question` | `string` | The original question as received by the server |
| `response` | `object \| null` | Parsed structured output from Gemini (see below) |
| `raw_response` | `string` | Unprocessed model output string |
| `plan` | `object \| null` | Planner stage output: situation summary, emotional tone, likely needs, action outline |
| `retrieved_kurals` | `array` | RAG retrieval results with `kural_id` and `score` |

The `response` object contains:

| Field | Type | Description |
|-------|------|-------------|
| `summary` | `string` | One-sentence framing of the response |
| `kurals` | `array` | Matched Kural objects with `kural_id`, `tamil`, `english`, `virtue`, `theme` |
| `actions` | `object` | Key-value map of action labels to prose guidance |
